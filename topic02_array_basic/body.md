# トピック2：1次元配列の基本（最大・最小・合計・平均・入れ替え）

## ①理論 ─ まず一言で（INPUT）
配列とは「番号つきのロッカー」。同じ種類のデータを一列に並べ、番号（添字）で1つずつ取り出せる。

なぜ必要？ 変数を100個作るのは非現実的。配列なら「A[i] の i を変えるだけ」でループと組み合わせ大量データを一括処理できる。トピック1の“ためる箱”を“棚”に拡張。

最大値の考え方：先頭を仮のチャンピオンにし、順に挑戦者と比較して勝者を残す（トーナメントの勝ち抜き）。合計・平均は1回走査：O(n)、空間 O(1)。

🗣️ 30秒で説明：「配列は番号つきの棚。最大値探しは“今の暫定王者と次を比べて強い方を残す”をラストまで繰り返すだけ」

## ②コード比較（例：最大値を求める）

全商・DNCL形式（添字1始まり）

    max ← A[1]
    i を 2 から 要素数 まで 1 ずつ増やしながら繰り返す:
        もし A[i] > max ならば:
            max ← A[i]
    を表示する: max

Python（添字0始まり）

    mx = A[0]
    for i in range(1, len(A)):
        if A[i] > mx:
            mx = A[i]
    print(mx)

🗣️ 説明チェック：DNCLはA[1]、PythonはA[0]から。同じ結果になる理由は？→「どちらも先頭を暫定王者にして2番目以降と比べる。番号の振り方が違うだけ」

## ⑤指導への還元
- 躓きポイント：0始まり/1始まりの混同。max を A[0] でなく 0 で初期化して負の配列で誤答する典型。
- キラーフレーズ：「最大値の初期値は“0”じゃなく“配列の先頭”にしろ」。
- AP発展：最大と最小を「2回走査」と「1回で同時取得」で比較。比較回数削減（3n/2 手法）に触れよ。
