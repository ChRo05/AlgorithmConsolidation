# トピック3：文字列処理（走査・カウント・回文判定・パターン照合）

## ①理論 ─ まず一言で（INPUT）
文字列とは「文字を一列に並べた配列」。トピック2の“番号つきの棚”の中身が文字になっただけで、S[i] で i 番目の1文字を取り出せる。

なぜ必要？ 名前・住所・URL・DNA…現実のデータの多くは文字列。数値と同じ“添字＋ループ”の型がそのまま使え、「1文字ずつ見て、数える／比べる」だけで検索も判定もできる。

計算量：端から端まで1回走査で O(n)。回文判定は左右を対で見るので実質 n/2 回比較の O(n)。素朴なパターン照合（テキスト長 n・パターン長 m）はずらしながら毎回照合し直すので O(n·m)。空間 O(1)。

🗣️ 30秒で説明：「文字列は“文字の配列”。i 番目と n−1−i 番目は鏡写しで、これを中央まで突き合わせれば回文か分かる。“1文字ずつ数える・比べる”の型は数値配列と同じ」

## ②コード比較（例：回文判定）

全商・DNCL形式（添字1始まり）

    n ← Sの文字数
    kaibun ← True
    i を 1 から n ÷ 2 まで 1 ずつ増やしながら繰り返す:
        もし S[i] ≠ S[n − i + 1] ならば:
            kaibun ← False
    を表示する: kaibun

Python（添字0始まり）

    n = len(s)
    kaibun = True
    for i in range(n // 2):
        if s[i] != s[n - 1 - i]:
            kaibun = False
    print(kaibun)

🗣️ 説明チェック：DNCLは対が (i, n−i+1)、Pythonは (i, n−1−i)。なぜズレる？→「1始まりは末尾が n、0始まりは n−1。鏡の相手の番号が1つずれるだけ」

## ⑤指導への還元
- 躓きポイント：文字列の添字も0始まり/1始まりで混同。len(S) と最終添字 n−1 を取り違えて範囲外参照。
- キラーフレーズ：「i 番目と n−1−i 番目は鏡写し」「照合はどこまでずらせるか＝ len(text)−len(pat) まで」。
- AP発展：素朴照合 O(n·m) から KMP・ラビン–カープの発想（ずらしの再利用／ハッシュ）へ。
