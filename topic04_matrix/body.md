# トピック4：2次元配列（行・列の走査、集計、二重ループ）

## ①理論 ─ まず一言で（INPUT）
2次元配列とは「座席表」。行（横の並び）と列（縦の並び）の2つの番号で、A[i][j]＝「i 行 j 列の席」を指す。1次元配列（棚）を縦に積んだもの。

なぜ必要？ 成績表・時間割・画像のピクセル・盤面ゲーム…現実のデータは「表」の形が多い。行番号と列番号の2つで指せれば、二重ループで表全体を1マスずつ処理できる。

走査の型：外側ループが行（i）、内側ループが列（j）。「外側＝行をめくる、内側＝その行の中を読む」。n 行 m 列の全走査は O(n·m)、空間 O(1)。

🗣️ 30秒で説明するなら：
「2次元配列は座席表。A[i][j] は“i 行目の j 番目の席”。処理は二重ループで、外の輪が行を1つずつめくり、内の輪がその行を左から右へ読む。これだけで表全体を1マスずつ触れる」

## ②コード比較（例：n 行 m 列の全要素の合計）

全商・DNCL形式（添字1始まり）

    goukei ← 0
    i を 1 から n まで 1 ずつ増やしながら繰り返す:
        j を 1 から m まで 1 ずつ増やしながら繰り返す:
            goukei ← goukei + A[i, j]
    を表示する: goukei

FE科目B形式

    整数型: goukei ← 0
    for (i を 1 から n まで 1 ずつ増やす)
        for (j を 1 から m まで 1 ずつ増やす)
            goukei ← goukei + A[i][j]
        endfor
    endfor
    return goukei

Python（添字0始まり）

    goukei = 0
    for i in range(len(A)):          # 行の数だけ
        for j in range(len(A[i])):   # その行の列の数だけ
            goukei += A[i][j]
    print(goukei)

🗣️ 説明チェック：Pythonで内側が range(len(A)) ではなく range(len(A[i])) なのはなぜ？
→「len(A) は“行の数”。行の中の席の数（列数）は len(A[i])。正方形の表なら偶然一致するが、長方形の表では別物」

## ⑤指導への還元（教員用メタノート）
- 躓きポイント：i＝行・j＝列の取り違え（A[j][i] と書いてしまう）。len(A)（行数）と len(A[0])（列数）の混同。正方行列のテストだけ通って長方形で壊れる“隠れバグ”。
- キラーフレーズ：「外側＝行をめくる、内側＝行の中を読む」「行き先に迷ったら“何行目の何番目”と声に出せ」「正方形で試すな、長方形で試せ」。
- 図解指示：実際の座席表（3行2列など非正方形）を描かせ、二重ループの進む順に 1,2,3… と番号を振らせる。行優先の“Z字”の動きを矢印で可視化。
- AP発展：行優先/列優先の走査順とキャッシュ効率（メモリ上は1列に並んでいる）を論ぜよ。転置（A[i][j]↔A[j][i]）の実装と、正方/非正方での違いに触れよ。
