# トピック1：規則的な計算（合計・カウント・フラグ・素数判定・最大公約数）

## ①理論 ─ まず一言で（INPUT）
規則的な計算とは「同じ手順を繰り返し、答えを1つの箱にためていく」こと。レジで商品を1つずつスキャンして合計金額をためる操作と同じ。

なぜ必要？ コンピュータは「一気に全部」ではなく「1つずつ順番に」しか処理できない。だから“ためる箱（変数）”と“繰り返し（ループ）”が全アルゴリズムの土台になる。

計算量：合計・カウントは1回走査で O(n)、空間 O(1)。素数判定は素朴に O(n)、√n打ち切りで O(√n)。最大公約数はユークリッドの互除法で O(log(min(a,b)))。

🗣️ 30秒で説明：「合計やカウントは、空の箱を用意して要素を順に見ながら箱の中身を更新する操作。この“箱＋繰り返し”という型が、後の探索も整列も全部の基礎になる」

## ②コード比較（例：1〜N の合計）

全商・DNCL形式

    goukei ← 0
    i を 1 から N まで 1 ずつ増やしながら繰り返す:
        goukei ← goukei + i
    を表示する: goukei

Python

    goukei = 0
    for i in range(1, N + 1):
        goukei += i
    print(goukei)

🗣️ 説明チェック：DNCLの「N まで」とPythonの range(1, N+1) が同じ範囲になる理由は？→「DNCLは終わりの数を含む、Pythonは含まない。だから+1する」
